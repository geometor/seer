SEER Process Workflow Overview

An agent designed to solve ARC-like tasks

Objectives:

  • Understand problem context
  • Describe problem & process
  • Convert language to code
  • Facilitation and coaching

I. Initialization and Setup

1. Configuration Loading

  • Reads config.yaml
  • Sets output directory
  • Defines models

2. Task Loading

  • Creates Tasks object
  • Loads .json tasks
  • Defines Task structure

3. Seer Creation

  • Creates Seer object
  • Initializes API clients
  • Loads instructions

4. Session Initialization

  • Creates Session
  • Creates directories
  • Writes config.yaml

II. Task Solving Loop (Outer)

1. Iterate through Tasks

  • Seer.run iterates

2. Task Setup

  • Sets directory
  • Initializes counter

3. Solve Task

  • Calls Seer.solve

III. Task Solving (Inner)

1. Investigate Examples

  • Iterates through pairs
  • Uses Dreamer & Builder

Natural Language (Dreamer)

  • Constructs prompt
  • Calls _generate
  • Processes response

Code Generation (Builder)

  • Constructs prompt
  • Calls _generate
  • Processes response

Response Processing

  • Extracts text & code
  • Saves/Executes code
  • Extracts results

Code Execution

  • Runs transform
  • Compares output
  • Logs results

2. Review Programs

  • Placeholder
  • Summarizes observations

IV. Generation

1. Prompt Construction

  • Combines inputs
  • Logs prompt

2. Model Interaction

  • Calls Gemini API
  • Handles retries

3. Response Processing

  • Calls _process_response

V. Code Execution

1. Execution

  • Parses & executes code
  • Captures output

2. Validation

  • Calls transform
  • Compares to expected
  • Logs results

3. Error Handling

  • Catches exceptions
  • Logs errors

4. Output & Logging

  • Captures output
  • Writes to file

VI. Session Management

1. Initialization

  • Creates directories
  • Writes config

2. Logging

  • Prompts & responses
  • Errors

3. Display

  • Config & results

VII. Helper Functions

  • File writing
  • Formatting
  • (Function Calls - Unused)

Key Considerations

  • Separation of Concerns
  • Logging & Error Handling
  • Code Validation
  • Extensibility
